Prerequist: init customer and models
    ### login to Robot VM
	$ export ROBOT_IP=<robot VM IP>
	$ ssh -o StrictHostKeychecking=no -i ~/.ssh/onap_key ubuntu@$ROBOT_IP
	$ sudo /opt/mydemo.sh init

1, Create complex: clli3, with latitude/longitude: 10/20

	### login to Robot VM
	$ sudo /opt/mydemo.sh add_complex clli3 10 20

2, on-board/register the eduge cloud to ONAP as a cloud region (CloudOwner/sx1)

	### using ESR GUI Portal, on-board a cloud region: (CloudOwner/sx0, tenant name: edge1)

		a, browse URL: http://msb.api.simpledemo.onap.org/iui/aai-esr-gui/extsys/vim/vimView.html
		
		b, Click "Register" button on the top left area
		
		c, Populate following fields as below data over the dialog page:
			Cloud owner:  CloudOwner
			Cloud region id:  sx0
			Cloud Type: openstack
			Cloud Region Version: titanium_cloud
			Physical location id: clli3
			Auth url: http://10.12.25.2:5000/v3
			User: demo
			password: <your password>
			Ssl-insecure: true
			Cloud Domain: Default
			Default Tenant: edge1
		
		d, Click "Save" button
			The dialog takes a few minutes to discover cloud region resource and then disappears

4, Add customer:
	### login to Robot VM
	### using robot script, add customer democust1 with service type: vLB
	sudo /opt/mydemo.sh add_customer democust1 vLB

5, Associate the cloud region (CloudOwner/sx0, tenant name: edge1) with customer (democust1)

	### login to Robot VM
	### using robot script, associate the customer with the on-boarded cloud region (CloudOwner/sx0) and tenant name "edge1"
